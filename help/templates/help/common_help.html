Цель сервиса

Основное назначение этого сервиса - помочь Web-разработчику создать готовый шаблон сайта, который можно впоследствии установить на систему управления содержимым. Входные данные - HTML файлы, результат - архив с шаблоном.

Адаптация под CMS осуществляется при помощи плагинов (модулей), который может быть написан под любую CMS, включая различные версии одной и той же CMS.

Поддерживаемые CMS

Список поддерживаемых систем управления контентом:

  • WordPress 4.6.1
  • Joomla 3.6.2

Входной файл

Входной файл представляет собой ZIP-архив с готовым сайтом на HTML. Подробнее на вкладке "Требования"

Список требований к проекту

Входные файлы, содержащиеся в архиве должны удовлетворять следующим требованиям:

  • Обязательно наличие файла index.html
  • Объекты для распознавания системой задаются при помощи атрибута ID у элемента шаблона

Ключевые ID задаются по определенным правилам:

  • ID должен быть указан у блочного элемента
  • Префикс page обозначает принадлежность элемента к общему содержанию страницы
  • Через дефис ставится название элемента

Примеры задания ID для элемента: <header id="page-header"></header> или <div id="page-footer"></div>

Примеры других ID: #page-header, #page-footer, #page-nav
